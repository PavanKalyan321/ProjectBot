================================================================================
TROUBLESHOOTING
================================================================================

Q: Dashboard shows "Loading..."
A: Make sure Flask is running on port 5001
   Check: python run_dashboard.py --port 5001

Q: "Disconnected" indicator (red)
A: WebSocket connection failed
   Check Flask is running and accessible
   Check .env.local has correct URLs

Q: No models showing
A: Need data from Flask backend
   Wait for at least one round to complete
   Check Flask console for errors

Q: Port 3000 already in use
A: Kill existing process or use different port
   npm run dev -- -p 3001

Q: "Cannot find module" errors
A: Run npm install again
   cd c:\Project\dashboard-nextjs
   npm install

IMPORTANT NOTES
================================================================================

1. Keep Flask running
   - One terminal for Flask backend
   - One terminal for Next.js frontend
   - Both must run simultaneously

2. Port Numbers
   - Flask: http://localhost:5001
   - Next.js: http://localhost:3000
   - Don't change unless necessary

3. Environment Variables
   - .env.local = Development
   - Set environment variables in production

4. Firewall
   - Windows may ask for firewall access
   - Allow both Flask and Node.js

5. Data Files
   - Flask reads CSV files from backend folder
   - Ensure CSV files exist for bot data

6. Database
   - Supabase integration also available
   - Can use CSV or PostgreSQL
