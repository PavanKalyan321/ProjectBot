COMMON QUESTIONS
================================================================================

Q: How do I connect to Supabase?
A: The connection is automatic! Just run your bot and it logs to Supabase.
   (Or run: python migrate_to_supabase.py to test)

Q: What happens to my CSV files?
A: They still work! But Supabase is now the primary database.
   Data is logged to BOTH by default (CSV + Supabase).

Q: Can I use this on another computer?
A: Yes! Just copy the c:\Project folder (including venv) to the new computer.
   The virtual environment will work on the same OS (Windows).

Q: What if I want to add packages later?
A: Easy! With venv activated, run: pip install package_name
   Then save: pip freeze > requirements.txt

Q: How do I install the full ML packages?
A: See VIRTUAL_ENV_QUICK_SETUP.txt - Option B (requires C++ Build Tools)

Q: What if something breaks?
A: Just delete the venv folder and run: python -m venv venv
   Then reinstall: pip install -r requirements.txt

Q: Is my data safe?
A: Yes! Supabase has automatic backups and SSL encryption.

Q: What's in the .env file?
A: Your Supabase credentials. KEEP IT SECRET! It's in .gitignore.

================================================================================
TROUBLESHOOTING
================================================================================

Problem: (venv) prefix not showing
Solution:
  1. Make sure you're in c:\Project folder
  2. Type: venv\Scripts\activate.bat
  3. Check for (venv) in prompt

Problem: "python: command not found"
Solution:
  1. Verify venv is activated (look for (venv) prefix)
  2. If not, run: venv\Scripts\activate.bat
  3. Try again: python --version

Problem: Connection timeout to Supabase
Solution:
  1. This is normal if direct connections timeout
  2. Use Supabase SQL Editor instead (recommended)
  3. Or wait a few moments and retry

Problem: Packages won't install
Solution:
  1. Use requirements-minimal.txt (already done)
  2. If errors about C++, see VIRTUAL_ENV_QUICK_SETUP.txt
  3. Message us if it persists

Problem: "venv folder missing"
Solution:
  1. Create it: python -m venv venv
  2. Install packages: pip install -r requirements.txt
  3. Takes ~2 minutes

For more help, see the detailed guides above!
